# Stolio — Design Document

## 1. Design Purpose

The purpose of this design document is to define how Stolio will present its product experience to users. It outlines the user journey, interface structure, component system, visual styles, and interaction patterns used in the platform.

The goal of the Stolio design system is to create portfolio websites that feel **personal, animated, and story-driven** while maintaining performance and simplicity. Instead of allowing full drag-and-drop editing like traditional website builders, Stolio will assemble portfolio pages using a structured set of reusable components. These components will be visually styled through different design systems such as Glassmorphism, Neobrutalism, Y2K Web, Clay UI, and Minimal Developer.

The design approach prioritizes clarity, storytelling, and performance. Each portfolio page will guide the viewer through a person’s professional story rather than presenting information in static blocks.

---

# 2. Design Principles

The Stolio interface follows several core design principles.

### Story First

Every generated portfolio must feel like a narrative journey rather than a list of information. Sections should guide visitors through the user's background, skills, and work in a natural progression.

### Structured Flexibility

Users should not need to manually design layouts. Instead, structured components will adapt to the user's data and chosen visual style.

### Visual Identity

Each style theme should produce a visually distinct portfolio. However, the underlying structure should remain consistent so the system remains scalable.

### Performance

Animations should enhance storytelling but must not degrade performance. Lightweight motion libraries and optimized rendering will be used.

### Simplicity

The editing process must remain simple. Users should update their portfolio through prompts or structured fields instead of complex layout tools.

---

# 3. Core User Journey

The Stolio user experience follows a simple flow designed to minimize friction.

### Step 1 — Landing Page

The user arrives at the Stolio landing page and learns how the platform works. The page introduces the concept of AI-generated storytelling portfolios and encourages the user to begin creating their portfolio.

Primary action: **Create Portfolio**

---

### Step 2 — Authentication

The user signs up or logs in to their Stolio account.

Supported authentication methods may include:

• Email and password
• Google login (future enhancement)

Accounts are required because portfolios must be stored and edited later.

---

### Step 3 — Portfolio Creation

The user begins the portfolio creation process.

Two input methods are provided:

Resume Upload
Users upload a PDF or DOCX resume.

Text Input
Users paste their resume or professional summary.

The system extracts relevant information and prepares it for AI processing.

---

### Step 4 — Portfolio Generation

After submitting their input, the system processes the information using AI.

The AI performs several tasks:

• extracts structured data
• generates a tagline
• writes a short bio
• organizes projects and skills
• builds the portfolio structure

Once complete, a preview of the portfolio is generated.

---

### Step 5 — Style Selection

Before finalizing the portfolio, the user chooses a visual style.

Available styles include:

Minimal Developer
Glassmorphism
Neobrutalism
Y2K Web
Clay UI

Users also choose theme preference:

Light mode only
Dark mode only
Mixed light and dark

The selected style is applied to the portfolio preview.

---

### Step 6 — Preview and Editing

Users can review their generated portfolio.

They can make changes through:

• prompt editing
• updating structured fields

Example prompt:

“Rewrite my about section to sound more confident.”

The system updates the portfolio content accordingly.

---

### Step 7 — Publishing

Once satisfied, the user publishes their portfolio.

The system generates a public link:

```
stolio.site/username
```

This link can be shared publicly.

---

# 4. Portfolio Page Structure

Each portfolio follows a consistent structure composed of reusable sections.

---

## Hero Section

The hero section introduces the portfolio owner.

Content includes:

• name
• professional title
• tagline generated by AI
• quick introduction
• optional social links

The hero section may include animated elements depending on the chosen style.

---

## About Section

This section tells the user’s professional story.

It includes:

• background summary
• career motivation
• major areas of focus

This section is written by AI but can be edited by the user.

---

## Skills Section

The skills section highlights the user’s technical or professional capabilities.

Skills are displayed through visual components such as:

• badges
• cards
• animated lists

The exact design depends on the selected style.

---

## Projects Section

The projects section showcases the user’s work.

Each project card includes:

• project name
• short description
• technologies used
• links (GitHub or demo)

Projects may be displayed using grids or stacked cards.

---

## Timeline Section

This section presents the user’s professional journey.

It may include:

• education
• internships
• work experience

The timeline format allows viewers to understand the user’s progression.

---

## Contact Section

The final section provides ways to connect with the user.

Typical links include:

• email
• LinkedIn
• GitHub
• portfolio downloads

This section encourages professional outreach.

---

# 5. Component System

Stolio portfolios are built using reusable UI components.

Each section is implemented as a component that receives structured data and renders it visually.

Example component structure:

HeroSection
AboutSection
SkillsSection
ProjectsSection
TimelineSection
ContactSection

Each component supports multiple visual variations depending on the selected style.

---

# 6. Visual Style System

Stolio supports five visual styles in the MVP.

Each style defines:

• color palette
• typography
• component appearance
• motion behavior

---

## Minimal Developer

Clean layout focused on readability.

Features:

• neutral color palette
• minimal animations
• developer-friendly typography

---

## Glassmorphism

Uses frosted glass panels and background blur effects.

Features:

• translucent panels
• soft shadows
• subtle gradients

---

## Neobrutalism

Bold and playful style.

Features:

• thick borders
• bright colors
• exaggerated shadows

---

## Y2K Web

Retro aesthetic inspired by early internet design.

Features:

• metallic gradients
• neon colors
• experimental typography

---

## Clay UI

Soft three-dimensional design.

Features:

• rounded shapes
• soft shadows
• layered cards

---

# 7. Animation System

Animations enhance storytelling but remain lightweight.

The animation system uses scroll-triggered and interaction-based animations.

Examples include:

• section fade-ins
• card hover animations
• subtle motion transitions between sections

Animations must remain smooth and not interfere with readability.

---

# 8. Editing Interface

The editing interface is intentionally minimal.

Users can modify their portfolio through:

Prompt Editing
Users describe the change they want.

Field Editing
Users directly update text fields.

Example editable fields:

• bio
• tagline
• project descriptions
• skills list

The system regenerates affected sections when edits occur.

---

# 9. Responsive Design

All portfolio pages must support responsive layouts.

Breakpoints include:

Desktop
Tablet
Mobile

Key adjustments include:

• stacked sections on smaller screens
• simplified navigation
• reduced animation intensity on mobile devices

---

# 10. Accessibility

The Stolio interface must follow accessibility guidelines.

Key requirements include:

• readable font sizes
• sufficient color contrast
• keyboard navigation support
• alt text for images

---

# 11. Performance Considerations

Portfolio pages must load quickly and remain responsive.

Optimization strategies include:

• static site generation where possible
• optimized animation libraries
• lazy loading for assets

These practices ensure portfolios remain fast even with visual effects.

---

# 12. Design Deliverables

The design system must include the following assets.

Component library
Visual style guidelines
Animation patterns
Responsive layout specifications

These deliverables allow developers to implement consistent portfolio pages.
